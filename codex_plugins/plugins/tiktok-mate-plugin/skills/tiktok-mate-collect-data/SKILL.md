# TikTok Mate 数据采集

## 适用场景
- 获取 TikTok 账号的个人资料（简介、粉丝数、点赞数）
- 获取当前视频的详细数据（播放量、点赞数、评论数、分享数）
- 批量采集多个账号或视频的数据
- 将采集结果整理为结构化数据

## 可用采集功能

| 功能 | 返回数据 | 说明 |
|------|----------|------|
| `getAccountInfo` | 账号资料（简介、粉丝数、点赞数等） | 需先导航到目标账号页面 |
| `getItemInfo` | 当前视频详情（播放、点赞、评论、分享） | 需先导航到目标视频 |

## 流程步骤

### 1. 单次数据采集

**获取账号信息**：
```
1. 进入 Feed 页 → 等待完成
2. 采集账号资料 → 等待完成 → 获取结果
```

**获取视频信息**：
```
1. 进入 Feed 页 → 等待完成
2. 采集当前视频详情 → 等待完成 → 获取结果
```

### 2. 批量数据采集

遍历多个视频：
```
1. 进入 Feed 页 → 等待完成
2. 循环:
   a. 采集当前视频数据 → 等待完成 → 保存结果
   b. 切换到下一个视频 → 等待完成 → 回到步骤 a
```

### 3. 数据整理

采集完成后，将结果整理为结构化格式：

**账号数据包含**：
- 个人简介 (bio)
- 粉丝数 (followers)
- 关注数 (following)
- 获赞数 (likes)
- 用户名 (username)
- 昵称 (nickname)

**视频数据包含**：
- 播放量 (playCount)
- 点赞数 (likeCount)
- 评论数 (commentCount)
- 分享数 (shareCount)
- 收藏数 (favoriteCount)
- 标题 (title)
- 时长 (duration)

## 最佳实践

1. **先导航再采集**: 采集数据前确保已进入目标页面
2. **采集后切换**: 批量采集时每个视频采集完后切换到下一个
3. **结果暂存**: 将采集结果保存到本地，防止数据丢失
4. **容错处理**: 单个采集失败不应中断整个批次

## 故障排查

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 数据为空 | 未导航到目标页面 | 先进入 Feed 页或对应页面 |
| 采集到错误数据 | 页面状态不对 | 确认当前页面内容是否符合预期 |
| 批量中断 | 单个操作失败 | 查看错误信息，跳过失败项继续 |
