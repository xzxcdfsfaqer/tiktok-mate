# TikTok Mate 服务状态查看

## 适用场景
- 检查服务是否正常运行
- 查看当前有哪些账号在线
- 查看可用的功能列表
- 诊断连接问题
- 查看服务已运行时间

## 流程步骤

### 1. 基础状态检查

调用 `alive` 功能查看服务状态：

```
alive()
```

返回信息包含：
- `uptime` — 服务已运行的秒数
- `availableAccounts` — 当前在线账号列表
- `availableJobs` — 可用的功能列表

### 2. 解读返回数据

| 信息 | 正常状态 | 异常状态 |
|------|----------|----------|
| 运行时长 | > 0 | ≤ 0（刚重启或异常） |
| 在线账号 | 包含预期账号 | 为空（无浏览器扩展连接） |
| 可用功能 | 包含 7 项功能 | 为空（服务初始化异常） |

### 3. 健康状态判断

```
if alive 调用成功:
  if 有在线账号:
    ✅ 服务正常，有账号在线
  else:
    ⚠️ 服务正常，但无在线账号
      → 检查浏览器扩展是否已启动
      → 检查扩展能否连接到服务
else:
  ❌ 服务不可达
    → 检查服务是否已启动
    → 检查编辑器中的服务配置是否正确
    → 检查端口 33001 是否被占用
```

### 4. 持续观察

如需持续观察服务状态，可每隔 30 秒调用一次 `alive`：

```
观察要点:
- 运行时长是否持续增长
- 在线账号数量和名单变化
- 是否有新账号上线或旧账号离线
```

**常见配置问题**:
- 地址拼写错误
- 端口号不是 33001
- 协议不是 http://
- 配置在错误的层级

## 诊断快查表

| 症状 | 可能原因 | 解决方法 |
|------|----------|----------|
| 无法连接 | 服务未启动 | 在终端运行 `node server.js` 启动服务 |
| 无在线账号 | 浏览器扩展未连接 | 检查浏览器中扩展是否已激活 |
| 账号突然离线 | 浏览器页面关闭 | 检查是否关闭了 TikTok 页面 |
| 运行时长很小 | 刚重启过 | 检查服务进程是否被意外终止 |
| 操作一直不执行 | 无在线账号 | 调用 `alive` 确认有在线账号 |

## 启动服务

如需启动服务：

```bash
node server.js
```

启动成功后日志会显示服务已就绪。
