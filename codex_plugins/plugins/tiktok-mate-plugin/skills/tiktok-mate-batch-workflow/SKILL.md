# TikTok Mate 批量自动化工作流

## 适用场景
- 编排多步骤的自动化流程
- 批量采集 Feed 中多个视频的数据
- 批量对多个视频执行点赞/收藏/关注
- "采集 → 互动"组合流程
- 定时/定期执行重复性操作

## 常见工作流模式

### 模式 1: 纯采集流水线

```
进入 Feed → 采集视频1数据 → 下一个 → 采集视频2数据 → 下一个 → ... → 汇总
```

**适用**: 批量获取多个视频的播放量、点赞数等数据。

### 模式 2: 纯互动流水线

```
进入 Feed → 点赞视频1 → 收藏视频1 → 下一个 → 点赞视频2 → 关注创作者 → ...
```

**适用**: 自动化执行社交互动。

### 模式 3: 采集 + 互动组合流水线

```
进入 Feed → 采集视频1数据 → 点赞视频1 → 下一个 → 采集视频2数据 → 收藏视频2 → ...
```

**适用**: 既需要采集数据又需要执行互动。

### 模式 4: 指定账号操作

```
查看在线账号 → 选择目标账号 → 用指定账号采集资料 → ...
```

**适用**: 多账号环境中指定特定账号执行操作。

## 流程步骤

### 1. 工作流编排模板

```
步骤 1: alive → 确认目标账号在线
步骤 2: startJob(gotoFeed, account) → 进入 Feed
步骤 3: 循环 N 次:
  步骤 3a: 采集当前视频数据
  步骤 3b: 等待操作完成 → 保存结果
  步骤 3c: (可选) 点赞/收藏/关注等互动
  步骤 3d: 等待操作完成
  步骤 3e: 切换到下一个视频
  步骤 3f: 等待操作完成
步骤 4: 汇总所有结果 → 输出报告
```

### 2. 批量参数建议

| 参数 | 说明 | 建议值 |
|------|------|--------|
| 检查间隔 | 每次查看操作结果的间隔 | 1~2 秒 |
| 操作等待 | 每个操作之间的等待 | 视前一个操作完成时间而定 |
| 批量数量 N | 循环处理的视频数量 | 建议 10~50（避免风控） |
| 超时限制 | 单个操作最大等待时间 | 120 秒 |

### 3. 错误处理策略

```
对每个操作:
  发起 → 等待结果
  if 操作失败:
    if 可重试（如网络抖动）:
      记录 → 重新执行该操作 → 最多重试 2 次
    else:
      跳过 → 记录错误到报告 → 继续下一个操作
  else if 操作成功:
    保存结果 → 继续
```

### 4. 结果汇总

所有操作完成后，将结果汇总为报告，包含成功数、失败数、具体每个操作的结果等。

## 最佳实践

1. **先检查环境**: 启动工作流前先用 `alive` 确认系统和在线账号正常
2. **渐进式执行**: 先小批量（3~5 个）测试流程，确认无误后再扩大
3. **进度保存**: 每完成一个操作就将结果保存，防止中途失败数据丢失
4. **控制频率**: 避免短时间内大量操作触发 TikTok 限制
5. **账号隔离**: 不同账号的操作尽量分开执行，避免混淆

## 故障排查

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 工作流中途卡住 | 某个操作长时间未返回 | 设置超时，超时后跳过继续 |
| 批量结果不完整 | 部分操作失败 | 查看失败列表，可单独重试 |
| 操作过于频繁被限 | 触发风控 | 增加操作间隔，降低批量数量 |
